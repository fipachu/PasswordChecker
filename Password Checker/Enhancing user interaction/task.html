Description

Until now, the user has had to restart the program whenever they want to check a different password. That's not very convenient! In this stage, you'll modify the program to accept passwords from users until they wish to stop continuously. Additionally, you'll get a chance to understand the argsparse module and command-line arguments in Python, adding more flexibility and control over the program's execution.

This stage adds user-friendly polish and flexibility. It includes hashing, API requests, user interaction, and command-line control.

Objectives

Your program should accomplish the following in this stage:

  1. Continuously prompt the user to enter a password;
  2. Provide an option for the user to exit the program at any time;
  3. Allow the user to pass an optional command-line argument to specify whether they want to see the full hashed password in the output (for example, --show-hash).

The functionality (hashing the password, sending the request, processing the response) remains the same as in the previous stages.

Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.

Without command-line argument:

Enter your password (or 'exit' to quit): > mypassword123
Checking...
Your password has been pwned! This password appears 23123 times in data breaches.
Enter your password (or 'exit' to quit): > exit
Goodbye!

With command-line argument --show-hash:

Enter your password (or 'exit' to quit): > mypassword123
Your hashed password is: 1cf4c502ddd89b918c4bfefea76dadd590693b48
Checking...
Your password has been pwned! This password appears 23123 times in data breaches.
Enter your password (or 'exit' to quit): > exit
Goodbye!
