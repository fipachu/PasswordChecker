Description

Now that your program can send a request to the Have I Been Pwned API, it's time to process the response. The response will include a list of hashes that match the prefix in the request, along with a count of how many times each hash appears in the data breaches recorded by the API. In this stage, find the hash corresponding to the user's password in this list and display the count to the user.

We'll add padding to our requests to further improve the program's privacy. Padding helps to conceal the size of the actual request. It involves sending additional, unnecessary, and accurate data to mask what is being sent.

In our case, the Have I Been Pwned API supports padding by adding Add-Padding: true to the request's header.

Note: Padded entries always have a password count of 0 and can be discarded once received.

Objectives

In this stage, you should modify your program to accomplish the following:

  1. Prompt the user to enter their password;
  2. Hash the user's password using the SHA1 hashing algorithm;
  3. Send a request to the Have I Been Pwned API using the first five characters of the hashed password, and add padding to the request;
  4. Process the response from the API to find the complete hash in the list and retrieve the count;
  5. Display a message to the user indicating whether their password has been pwned, and how many times it appears in the data breaches.
Examples

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.

Example 1:

Enter your password: > mypassword123
Your hashed password is: 1cf4c502ddd89b918c4bfefea76dadd590693b48
Checking...
Your password has been pwned! The password "mypassword123" appears 23123 times in data breaches.

Example 2:

Enter your password: > anotherpassword456
Your hashed password is: 4f275aab52e30f25e76124dc4aa9cc731ef256ae
Checking...
Good news! Your password hasn't been pwned.
